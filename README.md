**_Architecture_**

**Compare and contrast the types of frontend development you used in your full stack project, including Express HTML, JavaScript, and the single-page application (SPA).**

Within my full stack project, I used both server rendered pages with Express and a single page application (SPA) built with Angular. The Express approach involved generating complete HTML pages on the server, which required full page reloads for navigation, resulting in a less fluid user experience. Conversely, the Angular SPA dynamically updates content within a single page, providing a smoother, more responsive interface without the need for page reloads. Angular also emphasizes reusable components and services, making the codebase more organized and easier to maintain. Overall, while Express offers simplicity, the SPA offers enhanced interactivity and a more seamless user experience. 

**Why did the backend use a NoSQL MongoDB database?**

The backend was built using a NoSQL MongoDB database to manage both administrator authentication and travel package information. This choice was made to ensure swift scalability, allowing for a growing number of users and trips without limitations. Additionally, it enabled real time data updates, eliminating the need for server restarts or page refreshes to reflect changes. 

**_Functionality_**

**How is JSON different from Javascript and how does JSON tie together the frontend and backend development pieces?**

JSON is a format designed for storing and transmitting data, whereas JavaScript is a scripting language used to implement logic and interactive features. JSON itself does not execute code; it simply models data in a structured way. In this project, the frontend sent requests to the backend and received responses formatted as JSON. This JSON served as the communication link between the Angular interface and the Express server. 

**Provide instances in the full stack process when you refactored code to improve functionality and efficiencies, and name the benefits that come from reusable user interface (UI) components.**

Throughout the development process, I improved the code structure by extracting common API functionalities into Angular services rather than maintaining them within individual components. Additionally, I subdivided extensive parts of the user interface into smaller, reusable components. These changes resulted in a cleaner codebase and enhanced scalability. Utilizing reusable components not only promotes consistency across the application but also simplifies future updates and maintenance. 

**_Testing_**

**Methods for request and retrieval necessitate various types of API testing of endpoints, in addition to the difficulties of testing with added layers of security. Explain your understanding of methods, endpoints, and security in a full stack application.**

I employed custom console logging functions alongside external tools like Postman to verify the responses generated by my API. The administrator dashboard used HTTP methods such as GET, POST, and PUT to interact with the MongoDB database through Mongoose. Additionally, I implemented login authentication for administrators to ensure that access to the admin panel remained secure and protected.
